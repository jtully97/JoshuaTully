What Are Application Programming Interfaces?

In simple terms an API is a messenger that takes requests and tells a system what you would like to do with those request, and then returns the response back to you.

Take this example, when you go to a restaurant and want to order food you make a request to a waiter. Shortly after your food arrives and your request is satisfied. Think of the waiter as the interface between you and the kitchen system. The waiter can be viewed as what an API does for applications.

Similarly, an API lists a bunch of operations that are available for developers to use, along with a description with details of what they do. The developer doesn’t necessarily need to know how, for example, an OS builds and presents a user with a; “Submit Order” dialog box. They just need to know that it’s available for use witin the app they are building.

What Is REST?

"REST, or REpresentational State Transfer, is an architectural style for providing standards between computer systems on the web, making it easier for systems to communicate with each other." -codeademy.com

Priciples Guiding REST

Client–server - REST architecture is composed of clients, servers, and resources, with that, requests are handled through HTTP.

Stateless - Each request from client to the server must contain the necessary information to understand the request, and cannot take advantage of any stored context on the server. Session state is therefore kept entirely on the client.

Cacheable - All resources should allow caching unless explicitly indicated that caching is not possible.

Uniform interface - Resources should be uniquely identifiable through a single URL, and only by using the available methods of the network protocol.

Layered system - Allows an architecture to be composed of hierarchical layers by constraining component behavior such that each component cannot “see” beyond the immediate layer with which they are interacting.

Code on demand - In most cases code is a server will send back static representations of resources in the form of XML or JSON. Although when essential servers can send back executable code.

All About Postman.

Postman is a development tool that helps to build, test and modify APIs. With over 5 million users each month and over 250 million APIs, Postman has become an essential tool in developing APIs. It has the ability to make various types of HTTP requests like, GET, POST, PUT, PATCH.

Postman allows for reliable services due to the ability to build your API before deploying your code. It helps speed up the development process by having front end and back end teams working in parallel. A spectacular feature within Postman is the ability to explore the API by sending it different kinds of data to see what values are returned, which we will demo down below. I like to view Postman as sort of a github for API development, it allows for great collaboration and sharing of APIs being deployed and developed.
