ECMAscript

What is ECMAscript?

ECMAscript actually is the primary language that Javascript is based on. Digging into Javascript you can see that Javascript is listed as a major implementation of ECMAscript. So Javascript actually comes from ECMAscript. ECMAscript is a standardized specification of JavaScript, developed by Brendan Eich of Netscape; Originally it was called Mocha, followed by LiveScript and then eventually settling as Javascript. So to break it down into simple terms Javascript is basically ECMAscript at its core, but builds upon it. other scripting languages like JScript, and ActionScript, all use ECMAscript as its core as well. A good real world comparison would be the three scripting languages previously stated are all diffrent cars but run on the same motor (ECMAscript).

The first edition of ECMA-262 was adopted by the Ecma General Assembley in June of 1997. The name "ECMAscript" was introduced by the organizations involved with standardizing the language, (Netscape & Microsoft). A little funny thing to mention is that Brendan Eich commented that "EchmaScript was always an unwanted trade name that sounds like a skin disease."

Currently there are 10 different versions of ECMA-262. Work on the 10th version was finalized in June of 2019, leading to ES.Next which is a dynamic name that refers to whatever the next version of ECMAscript is at the time of writting. The cool thing about ES.Next is that ES.Next features are finished proposals so viewers can in a way speculate about features that may be arriving to the standard with the next release.

Microservices vs Monolithic Architecture

Monolithic Architecture is set up for traditional server side systems. The entire system's function is based on a single application. This type of Architecture comes with some advatages the first being that it is rather fast to develop, an application can be developed with basic features and then scaled up over time. Monolithic apps can in some cases be faster than microservice apps because they do not have the need to communicate with various APIs. Every piece of software within a Monolithic system all communicate with the same Database and are located on the same server, they also will use the same file system. Monolithic applications in the long run can become difficult in maintaining because of their tightly coupled processes, where making a single change can create a domino effect causing problems throughout the entire application.

Microservice Architecture is an Architecture where every feature is it's own application. Things like authentification and posting are their own application on their own server but are connected through the use of APIs. connecting seperate applications in general is preformed through the use of JSON API based applications. Microservices are not nearly as tightly coupled as Monolithic applications, each services is not dependent on one another so making a single change wouldn't effect the other services because they are standalone. One of the major advatages of Microservices is their scalability, you can scale up individual componets instead of having to scale up the application as a whole. Since each component can function without the other, if one compnent fails the others don't.

THE MITT ROMNEY WEB FONT PROBLEM

The problem here is that multiple font weights and styles of the same typeface were used together so in the case of the mitt romney webfont problem the word "not" was italicized so that when the web page was loading the flash of invisble text was in the place of the word not making the headline read "Mitt Romney is officially running for president." This created a lot of probles when reading the article for the text outside the italicized text was displayed before the italicized text making readers beleive that Mitt Romney was officially running for president when infact he wasn't.

This problem can create confusion across a multitude of cases. One word can make a major diffrence in headlines and within text. failing at delivering proper information can cause credibility to diminish within a few views. All it take is for a developer to inappropriately apply a font and then the real issues happen when the mal practice gets noticed and captured by a user.

This can all be fixed in my personal opinion by testing a font out on multiple browsers. If there is a font and a case that must be used developers should take the steps to make sure that the font delivery doesn't run into discrepancies. Developer can eliminate the FOIT and switch it to a fallback font that loads while the selected font is loading. Also you can group your fonts so they load in a single repaint. One last way to get around this problem is through utilizing font-synthesis to show browser generated bold and italic fallbacks while the real web fonts load.

Nodejs and the V8 Javascritpt Engine

Node.js is an open-source and cross-platform JavaScript runtime environment. Node.js runs the V8 Engine. The V8 Engine is a powerful open source Javascript engine provided by Google. Now what exactly is a Javascript Engine? It is software that converts Javascript code into lower level or machine code that microprocessors can understand. This is essential to make sure that Javascript runs the same way acrosss multiple devices within diffrent browsers.

In this segement we are focusing on the V8 Javascript engine that is used within google chrome, but there are more than just the v8 Engine there are many for diffrent browsers. For example there are the Rhino, JavaScriptCore, and spider monkey. All of these engines follow the EchmaScript standards as discussed before, the EchmaScript standards define how the language should work and what features it should have.

The V8 Engine is written in in C++ and used within Chrome and Nodejs. What is cool is that the V8 Engine can run standalone within our own C++ program. What that means is we can use our own function implementation in C++ to add new features to Javascript. This creates a vast amount of flexibility, paired with unrivaled usability within JavaScript. I find the ability to create new JavaScript functionality extremely fascinating, this in its self helps propels web development into the future.
