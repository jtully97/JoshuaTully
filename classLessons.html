Functions

A JavaScript function is a block of code designed to perform a specific task.
A JavaScript function is executed when something invokes it aka calling it.

Named Functions

Named functions are functions that recieve a name upon creation.
They have several advantages to their counterpart (anonymous functions). one advatage is readability, since you have to name it, you end up having to make it more understandable to developers of all skill levels. They also are very useful for identifying what functions cause errors during development.
Named functions are also easier to reuse. When a function is named it makes it easier for developers to grasp what the exact function is doing and implement it for similar use cases.

Function Expression

A JavaScript function can also be defined using an expression.
A function expression is very similar to a function decleration, one of the biggest diffrences is how the browser loads them into the execution context.

Function declerations load before any code is executed while, function expressions load only when the interpreter reaches that line of code. So given the case where you try to call a function expression before it's loaded, an error will occur. Whereas if you call a function decleration it will work because hoisting occurs at run time.

Scope

JavaScript has two scopes, global and local. Any variable declared outside of a function belongs to the global scope and with that is accessible anywhere in the code. Local scope is created within functions, variables created within a function are part of the local scope.
