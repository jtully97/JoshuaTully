Prototypal Inheritance

Here we will discuss Prototypal Inheritance.

Attributes

In a HTML document there are attributes. Within the JavaScript DOM we can communicate to and change these attributes through properties and also use method calls that exist on elements, known as DOM nodes, when one makes a query. Attributes provide additional information about elements. These attributes are specefied within what is known as a start tag, they usually come in name/value pairs

Functions

Function instances inherit methods and properties from Function.prototype. Typical with all constructors, you can change a constructor's prototype object to make changes to all Function instances.

`this`, in the scope of inheritance

"When an inherited function is executed, the value of this points to the inheriting object, not to the prototype object where the function is an own property." -Mozilla.org

`this` refers to the object that is inherting the prototype attribute.

Syntactic Sugar

Makes code easier for humans to read and or write, but that do not change the functionality or expressiveness of the language. It sometimes is reffered to as making the language more "sweet" for human use, concise, easy to read and grasps understanding when looked at.

Window Object

The JavaScript window object has several properties and methods that can be used and invoked, but make sure to be carful for that fact that not all of them are supported in every browser. The window object is helpful in many ways, say you wan tot get the size of the users browser window, one would use the window object to obtainn this kind of information.

The window object has several properties. They include constructors, methods, and value properties. There are methods to manipulate the current browser tab, like opening and closing new pop-up windows, etc.

Global Object

A global object is an object that always exists in the global scope. Within JavaScript, there is always a global object that is defined. In a web browser, when scripts create global variables, they are created as part of the global object, with that the global object provides variables and functions that can be accessed anywhere.
